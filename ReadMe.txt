Идея обработки большого количества транзакций в данном проекте заключается в следующем: 
при последовательном считывании записей из таблицы транзакций базы данных (БД) они преобразуются в объекты Transaction. Каждый объект Transaction состоит из объектов TransactionItem.
TransactionItem это эквиваленты полей в строках таблицы БД, однако вместо хранения значения поля TransactionItem содержит хэш этого значения. Такой подход позволяет полностью абстрагироваться от содержимого полей транзакции. Для примера исходных данных о грибах (тестовое задание) эффективнее было бы хранить сами значения, т.к. каждое значение представлено всего одним символом. Но я решил сделать используя хеш.
Для разбиения транзакций по группам в проекте создан класс кластера Cluster, который имеет два метода AddTransaction и RemoveTransaction. Эти методы позволяют соответственно добавить и удалить информацию о транзакции в кластер. Однако внутри Cluster-а не хранятся сами транзакции, а хранятся экземпляры класа TransactionItemStatistic. Класс TransactionItemStatistics содержит информацию о количестве вхождений в кластер каждого элемента транзакций, т.е информацию о TransactionItem. Например если в кластер добавлены транзакции {a,d,f} и {a,с,e}, то сам кластер будет хранить только информацию об уникальных элементах, т.е. список вида: a=2, c=1, d=1, e=1, f=1. 
В проекте класс CLOPEAnalizing содержит статические методы для вычисления приростов для текущего кластера и текущей транзакции с учётом коэффициента отталкивания, а также хранит сам коэффициент отталкивания.
Такие классы как TransactionItemStatistics, ClusterRepository и DBHelper являются вспомогательными. TransactionItemStatistics описывает коллекицю экземпляров TransactionItemStatistic и автоматизирует процесс добавления транзакции в кластер.
ClusterRepository создавался как эквивалент хранилища кластеров с автоиндексацией, но в ходе разработки оказалось, что он не нужен. Но несмотря на невостребованность класс ClusterRepository не был удалён по причине возможного добавления в будущем кластеров в БД. В этом случае ClusterRepository будет выступать связующим звеном между БД и кодом.
DBHelper содержит вспомогательные методы для получения объектов работы с БД.
Само задание выполнено в WinForms проекте на .Net framework 4.6.2 платформе.
